Reference

CLI Reference

Complete reference for all synapse CLI subcommands. The CLI controls your local daemon, manages agents and environment variables, and authenticates with the Synapse cloud. New to Synapse? Start with the Getting Started guide.

Authentication

synapse login

synapse login [--cloud-url URL] [--timeout DURATION]

Start the RFC 8628 Device Authorization Grant flow. The CLI prints an 8-character user_code and a verification URL. Open the URL in your already-authenticated browser and approve the device. The CLI polls the cloud until the request is approved, then stores the access/refresh token pair securely in the OS keychain.

FlagDefaultDescription
--cloud-url URL https://synapse.cloud Override the cloud endpoint (useful for self-hosted deployments).
--timeout DURATION 5m Maximum time to wait for browser approval before giving up.
synapse login
$ synapse login
To authorize this device, visit:
  https://synapse.cloud/device
 
Enter code: XKCD-7A2F
 
Waiting for approval...
Waiting for approval.....
Authenticated as alice@example.com
Token stored in OS keychain.

Daemon

synapse init

synapse init [--name NAME] [--cloud-url URL]

Interactive wizard that creates ~/.synapse/daemon.toml. Generates an X25519 keypair (private key stored in the OS keychain, public key registered with the cloud), sets the daemon name, sets the cloud URL, and configures the default log level.

FlagDefaultDescription
--name NAME hostname Human-readable name for this daemon instance.
--cloud-url URL https://synapse.cloud Cloud endpoint to register with.
synapse init
$ synapse init
Synapse daemon setup wizard
 
Daemon name [my-macbook]: work-laptop
Cloud URL [https://synapse.cloud]:
Log level (debug/info/warn/error) [info]:
 
Generated X25519 keypair — private key stored in OS keychain
Public key registered with cloud
Config written to ~/.synapse/daemon.toml
 
Run `synapse daemon run` to start the daemon.

synapse daemon run

synapse daemon run [--config PATH] [--log-level LEVEL]

Start the daemon in the foreground. The daemon establishes an outbound WebSocket connection to the cloud uplink and begins processing agent runs dispatched by the cloud. Log output goes to stdout. Press Ctrl-C for a graceful shutdown — in-flight runs are allowed to finish before the process exits.

FlagDefaultDescription
--config PATH ~/.synapse/daemon.toml Path to the daemon configuration file.
--log-level LEVEL info Log verbosity: debug, info, warn, or error.
synapse daemon run
$ synapse daemon run
Starting Synapse daemon v0.9.4
Connected to cloud uplink wss://synapse.cloud/ws/daemon
Listening for agent runs... (Ctrl-C to stop)
2026-06-10 09:14:32 [INFO ] run:a1b2c3 agent=pr-reviewer started
2026-06-10 09:14:45 [INFO ] run:a1b2c3 agent=pr-reviewer finished in 13.2s

synapse daemon install

synapse daemon install [--user]

Install the daemon as a native system service so it starts automatically on boot. Uses launchd on macOS, systemd on Linux, and Windows Service on Windows. By default installs as a system-level service (requires administrator/root privileges). Pass --user to install as a user-level service instead.

FlagDefaultDescription
--user false Install as a user service (no root required) rather than a system service.
synapse daemon install
$ synapse daemon install --user
Installed user service: com.synapse.daemon
Service started
Daemon will start automatically on login.

synapse daemon status

synapse daemon status

Show the current daemon status: whether it is running or stopped, uptime, cloud connection state, number of active agent runs, and current memory usage.

synapse daemon status
$ synapse daemon status
Daemon is running
Uptime 2h 34m 11s
Cloud connected (wss://synapse.cloud)
Active runs 1
Memory 48.2 MB RSS

synapse daemon stop

synapse daemon stop [--force]

Gracefully stop the running daemon. The daemon waits for any in-flight agent runs to complete before exiting. Use --force to kill the process immediately without waiting for runs to finish.

FlagDefaultDescription
--force false Kill the daemon process immediately, abandoning in-flight runs.
synapse daemon stop
$ synapse daemon stop
Waiting for 1 in-flight run(s) to finish...
Daemon stopped

synapse daemon logs

synapse daemon logs [--follow] [--since DURATION] [--level LEVEL]

Print daemon log output. By default shows the tail of the current log. Use --follow to stream live output, --since to limit to recent time windows, and --level to filter by minimum severity.

FlagDefaultDescription
--follow / -f false Stream new log lines as they are written.
--since DURATION — Show logs from the last duration, e.g. 1h, 30m.
--level LEVEL info Minimum log level to display: debug, info, warn, error.
synapse daemon logs
$ synapse daemon logs --since 30m --level warn
2026-06-10 09:02:11 [WARN ] run:d4e5f6 tool=bash exit_code=1 retrying (1/3)
2026-06-10 09:12:03 [WARN ] uplink reconnect attempt 1 (backoff 2s)
Reconnected to cloud uplink

Agents

synapse agent list

synapse agent list [--json]

List all agents in the organization with their status (enabled/disabled), last run time, and total run count. Pass --json to receive machine-readable JSON output.

FlagDefaultDescription
--json false Output results as a JSON array instead of a human-readable table.
synapse agent list
$ synapse agent list
NAME STATUS LAST RUN RUNS
pr-reviewer enabled 2026-06-10 09:14 142
release-drafter enabled 2026-06-09 18:03 38
dep-scanner disabled 2026-05-21 11:22 19

synapse agent create

synapse agent create --name NAME --type TYPE [--prompt-file PATH]

Create a new agent in the organization. --type must be one of api-model (calls a language model API) or cli-tool (runs a local shell command). Use --prompt-file to load the system prompt from a Markdown file on disk.

FlagDefaultDescription
--name NAME — Unique slug for the agent (required).
--type TYPE — api-model or cli-tool (required).
--prompt-file PATH — Path to a Markdown file containing the system prompt.
synapse agent create
$ synapse agent create --name dep-auditor --type api-model --prompt-file ./prompts/dep-auditor.md
Agent "dep-auditor" created (type: api-model)
System prompt loaded from ./prompts/dep-auditor.md (1.4 KB)

synapse agent run

synapse agent run NAME [--env KEY=VALUE ...] [--wait] [--timeout DURATION]

Trigger a manual run of the named agent. By default the command returns the run ID immediately and the run executes asynchronously on the daemon. Pass --wait to block until the run completes and show the exit status. Use --env to override environment variables for this single run only (without modifying stored secrets).

FlagDefaultDescription
--env KEY=VALUE — Override an env var for this run only. May be repeated.
--wait false Block until the run finishes and print the result.
--timeout DURATION 10m Maximum time to wait when --wait is set.
synapse agent run
$ synapse agent run pr-reviewer --wait --env PR_NUMBER=482
Run enqueued: a1b2c3d4
Waiting for run a1b2c3d4 to complete...
Run a1b2c3d4 finished in 13.2s (exit: success)

synapse agent logs

synapse agent logs NAME [--run RUN_ID] [--follow] [--since DURATION]

Print or stream logs for an agent. Without --run, shows logs from the most recent run. Combine with --follow to tail an active run in real time.

FlagDefaultDescription
--run RUN_ID latest Show logs for a specific run ID rather than the most recent.
--follow / -f false Stream new log lines as they arrive.
--since DURATION — Limit output to the last duration, e.g. 5m.
synapse agent logs
$ synapse agent logs pr-reviewer --follow
2026-06-10 09:14:32 [INFO ] run started
2026-06-10 09:14:33 [INFO ] fetching PR #482 diff (4 files)
2026-06-10 09:14:41 [INFO ] model responded (718 tokens)
2026-06-10 09:14:44 [INFO ] posted 3 review comments
run finished in 13.2s

synapse agent schedule add

synapse agent schedule add NAME --cron "EXPR" [--tz TIMEZONE] [--overlap POLICY]

Add a cron schedule to an agent. The cron expression follows standard five-field syntax (minute hour day-of-month month day-of-week). The --overlap policy controls what happens when a new run is triggered while a previous run is still in progress.

FlagDefaultDescription
--cron "EXPR" — Standard cron expression, e.g. "0 9 * * 1-5" (required).
--tz TIMEZONE UTC IANA timezone for cron evaluation, e.g. America/New_York.
--overlap POLICY coalesce Overlap policy: coalesce (skip new run if one is running), run_once (queue at most one pending), or allow (always start a new run).
synapse agent schedule add
$ synapse agent schedule add dep-auditor --cron "0 8 * * 1" --tz America/New_York --overlap coalesce
Schedule added to "dep-auditor"
Cron 0 8 * * 1 (America/New_York)
Overlap coalesce
Next run 2026-06-15 08:00:00 EDT

Environment Variables

⚠

Values are always read from stdin, never passed as CLI arguments. This prevents secrets from appearing in shell history, process lists, or log files.

synapse env set

synapse env set AGENT_NAME KEY

Set an environment variable for an agent. The value is read from stdin — never passed as a CLI argument — and is encrypted client-side with the agent's X25519 public key before being uploaded. The cloud stores only the ciphertext; the plaintext value is never transmitted unencrypted and is never accessible server-side.

FlagDefaultDescription
No flags — Value is always piped via stdin.
synapse env set
# Pipe the secret value via stdin
$ echo "sk-proj-abc123..." | synapse env set pr-reviewer OPENAI_API_KEY
OPENAI_API_KEY encrypted and stored for agent "pr-reviewer"
 
# Or read from a file
$ cat ~/.secrets/gh-token | synapse env set release-drafter GITHUB_TOKEN
GITHUB_TOKEN encrypted and stored for agent "release-drafter"

synapse env list

synapse env list AGENT_NAME

List the names of environment variables configured for an agent. Values are never shown — they are write-only once stored. This command only reveals which keys exist.

synapse env list
$ synapse env list pr-reviewer
OPENAI_API_KEY set (updated 2026-06-10)
GITHUB_TOKEN set (updated 2026-05-28)
 
# Values are never displayed — only key names

synapse env delete

synapse env delete AGENT_NAME KEY

Permanently delete a stored environment variable for an agent. The ciphertext is removed from the cloud immediately.

synapse env delete
$ synapse env delete pr-reviewer OPENAI_API_KEY
Delete OPENAI_API_KEY from "pr-reviewer"? [y/N]: y
OPENAI_API_KEY deleted

Plugins

synapse plugin install

synapse plugin install PLUGIN_NAME [@VERSION]

Install a plugin or skill from the Synapse marketplace to the local daemon. Plugin names without a namespace are tool integrations (e.g. github). Prefix with skill: to install agent skills. Append @VERSION to pin a specific release.

FlagDefaultDescription
No flags — Version is optional; omit for the latest release.
synapse plugin install
$ synapse plugin install github
Installed plugin: github v2.1.0
 
$ synapse plugin install skill:security-scan@1.2.0
Installed skill: security-scan v1.2.0

synapse plugin list

synapse plugin list

List all installed plugins and skills with their current version and activation status.

synapse plugin list
$ synapse plugin list
NAME TYPE VERSION STATUS
github plugin 2.1.0 active
slack plugin 1.4.2 active
skill:security-scan skill 1.2.0 active
skill:code-review skill 0.9.1 disabled

synapse plugin remove

synapse plugin remove PLUGIN_NAME

Remove an installed plugin or skill from the local daemon. The daemon must be restarted for the change to take full effect.

synapse plugin remove
$ synapse plugin remove skill:code-review
Removed skill: code-review
Restart the daemon for changes to take effect.

TUI Dashboard

synapse tui

synapse tui

Launch the Textual-based terminal UI dashboard. The TUI shows live agent status, active runs with streaming log output, recent run history, and the human-in-the-loop approvals queue. All data is read from the local daemon over a Unix socket.

Keyboard shortcuts:

FlagDefaultDescription
No flags—No configuration flags; uses the default daemon socket.
KeyAction
qQuit the TUI
rForce refresh all panels
aApprove the selected item in the approvals pane
lJump to the logs pane
synapse tui
$ synapse tui
Connecting to daemon... connected

Version

synapse version

synapse version [--json]

Print version information for the local Synapse CLI/daemon, the Python runtime, the operating system, and the cloud backend the daemon is currently connected to.

FlagDefaultDescription
--json false Output version information as a JSON object.
synapse version
$ synapse version
Synapse daemon 0.9.4
Python 3.12.4
OS macOS 15.4.1 (arm64)
Cloud 0.9.4 (wss://synapse.cloud)

{
"daemon": "0.9.4",
"python": "3.12.4",
"os": "macOS 15.4.1 arm64",
"cloud": "0.9.4"
}
